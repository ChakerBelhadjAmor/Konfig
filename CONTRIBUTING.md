# Contributing to Konfig

Thank you for your interest in contributing!

By submitting a pull request you agree to the following:

- Your contribution becomes the intellectual property of Chaker Belhadj Amor
- You grant the author full rights to use, modify, and relicense your contribution under the current and any future license
- You confirm the contribution is your own original work
- You will not be compensated unless explicitly agreed upon in writing
- You have read and agree to the [LICENSE](LICENSE) file

## How to Contribute

1. **Open an issue first** — describe the change you want to make and why
2. Wait for approval before starting work to avoid wasted effort
3. Fork the repo (for development purposes only)
4. Create a focused feature branch (`fix/slider-bug`, `feat/alacritty-support`, etc.)
5. Make your changes — keep the scope tight and the diff readable
6. Open a Pull Request with a clear description of what changed and why

## What We Accept

- Bug fixes
- Performance improvements
- New terminal support
- UI/UX improvements
- Documentation improvements

## What We Do Not Accept

- Changes that break existing functionality without prior discussion
- Code copied from other projects without clear licensing
- PRs without a clear description
- PRs that haven't been discussed in an issue first

## Code Style

- Follow the existing code conventions in the project
- Keep components focused and avoid unnecessary abstractions
- Run the project locally and verify your changes work before opening a PR
